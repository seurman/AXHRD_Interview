# HR_IN 상용화 · 특허 회피 가이드

## 차별화 포인트 (자체 IP)

1. **IRT 2PL 적응형 엔진** — 공개 학술 이론(2PL IRT) 기반 자체 Python 구현. 역량별 θ 추정·문항 선택 로직은 HR_IN 독자 코드.
2. **역량 단위 세션 (Competency Mode)** — 한 번에 6역량 전체가 아니라 역량당 2~3문항 + 피드백 후 다음 역량. UX·데이터 모델 자체 설계.
3. **자소서 기반 질문 재작성** — 자체 문항 템플릿 + LLM/heuristic 개인화. 타사 면접 문항 DB·특허 청구항의 "고정 문항 세트"와 무관.
4. **레벨 칩(♩♪♭) 피드백** — IRT 결과를 음표형 칩으로 시각화하는 독자 UI 패턴.
5. **회원·플랜·역량 진행률** — PostgreSQL + (선택) Neo4j 온톨로지로 장기 트래킹.

## 회피 권장 영역

| 영역 | 회피 방법 |
|------|-----------|
| 타사 면접 문항 DB | 자체 seed 문항만 사용, 외부 문항 크롤링·라이선스 DB 미사용 |
| 영상·표정·음성 감정 AI | 텍스트/음성 STT + 루브릭 평가만. 감정·거짓말 탐지 미구현 |
| "한 세션 N문항 고정" 특허 | 역량별 조기 종료(2~3문항) + IRT 종료 규칙으로 설계 분리 |
| 제3자 적성검사 문항 | MMPI 등 표준화 검사 문항 미사용 |

## 기술 스택 (AWS 이전 대비)

- Web: Next.js 15, Prisma, JWT 세션
- DB: PostgreSQL (필수), Neo4j (그래프·온톨로지, 선택)
- IRT: FastAPI 마이크로서비스
- LLM: Gemini 2.5 Flash-Lite/TTS(평가·개인화·TTS), DeepSeek(피드백 리포트) — API 키 환경변수, 비중국 벤더로 통일(PIPA 국외이전 리스크 회피)
