# AXHRD 고객 시연 스크립트 (고정 2계정)

비밀번호 공통: **`Demo2026!`**

시드 적용:

- **로컬:** `cd web && npm run db:seed:demo`
- **운영(Vercel):** 슈퍼어드민 → `/admin/diagnostic` → **Campaign** 탭  
  · 「운영 통합 시연 시드」 — 개인·쇼케이스·지원자·쇼케이스 ARC (약 1~2분)  
  · 「운영 ARC 데모」 — 테크노바 + 쇼케이스 조직진단 풀 리포트 (통합 시드에 테크노바 미포함)

---

## 계정 A — 개인 코칭 (`dashboard-demo@demo.axhrd.local`)

**스토리:** 신입 마케터 김하은 — 5회 면접·6축·차수 요약·자기발견 데이터가 이미 쌓여 있음.

| 순서 | 화면 | 말할 포인트 |
|------|------|-------------|
| 1 | `/login` → 로그인 | 개인 Career OS 데모 계정 |
| 2 | `/dashboard` | 상단 **코칭 한 줄** — 최근 차수·약점 역량 요약 |
| 3 | 대시보드 스크롤 | 6축 레이더·시계열, 역량 숙련도, **차수 요약** 패널(한 줄 내러티브) |
| 4 | 코칭 인사이트 | **최근 면접·세션** 목록 → 리포트 링크 |
| 5 | 면접 리포트 1건 | **리포트 한 줄** + 강점·개선점 |
| 6 | `/interview/setup` | 자소서 업로드 시 **청크 단위 인용**으로 질문 생성 (데모 자소서는 이미 시드됨) |
| 7 | 면접 진행 중 | 질문 아래 **자소서 근거** 배지 — `[프로젝트명] 발췌` 형태 |

---

## 계정 B — 기관 풀패키지 (`arc-demo-admin@demo.axhrd.local`)

**스토리:** AXHRD 쇼케이스 기관 관리자 — 조직진단·코호트·지원자 스크리닝 모두 활성.

| 순서 | 화면 | 말할 포인트 |
|------|------|-------------|
| 1 | `/login` → 로그인 | 기관 관리자 — 진단 + 채용 스크리닝 |
| 2 | `/org/diagnosis` | 조직진단 웨이브·팀별 응답·ARC 리포트 |
| 3 | `/org/dashboard` | 코호트 역량 비교·기관 대시보드 |
| 4 | `/org/candidates` | 인터뷰 킷 공유 링크 + **3명 지원자** 완료 세션 |
| 5 | 캠페인 → **지원자 비교표** | 종합 점수·6축 히트맵으로 3명 스크리닝 비교 |
| 6 | 지원자 리포트 1건 | 채용 담당자 관점 스크리닝 리포트 |

---

## 슈퍼어드민 (선택)

환경변수 `SUPERADMIN_EMAIL` 계정으로 `/admin/diagnostic` → **테크노바 (ARC 데모)** 2웨이브 풀 리포트 (별도 로그인 계정 없음).
